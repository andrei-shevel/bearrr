Privacy Policy

Last updated: 22 July 2026

Mantella is a local-first desktop app. It doesn't have an account system, doesn't run analytics or telemetry, and doesn't send your documents, library contents, or reading activity anywhere. This page explains exactly what little network activity the app does have.

What Mantella collects

Nothing. Mantella has no servers of its own and no analytics or crash-reporting service is built into the app. There is no account to create and no personal information is ever requested.

What's stored, and where

Your PDF library, reading position, zoom level, scroll offset, and pins are stored in a local settings file on your own computer, alongside app preferences. None of this leaves your device, and Mantella has no way to read it remotely even if it wanted to.

Auto-updates

On startup, and whenever you use "Check for Updates…", Mantella makes an HTTPS request to GitHub to see if a newer release is available, and downloads the update from GitHub if you choose to install it. This is a plain request for a public release manifest — no document contents, library data, or identifying information about you is included. As with any web request, GitHub's servers will see standard connection details (such as your IP address and app version) as part of normal HTTPS traffic; Mantella doesn't add anything to that beyond what's needed to check for a new version. See GitHub's privacy statement for how they handle that traffic.

Third parties

Mantella doesn't embed ads, trackers, or third-party SDKs. The only external service it talks to is GitHub, solely for auto-updates as described above.

Changes to this policy

If Mantella's data practices ever change, this page will be updated to reflect that before the change ships.

Contact

Questions can be raised via the issue tracker.
